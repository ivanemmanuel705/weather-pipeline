# Weather Data Pipeline 🌦️

## Architecture
OpenWeatherAPI → EventBridge → Lambda → DynamoDB → DynamoDB Stream → Lambda → S3 → Snowpipe → Snowflake

## Components

### AWS Services
- **OpenWeather API**: Source of live weather data
- **EventBridge**: Triggers Lambda on a schedule (every minute/hour)
- **Lambda (1st)**: Fetches weather data → saves to DynamoDB + S3
- **DynamoDB**: NoSQL database storing weather records
- **DynamoDB Stream**: Captures changes in DynamoDB in real time
- **Lambda (2nd)**: Triggered by DynamoDB Stream → writes to S3
- **S3**: Stores raw JSON weather files
- **SQS**: Event notification queue → triggers Snowpipe when new file arrives
- **IAM**: Manages secure access between S3 and Snowflake

### Snowflake
- **Storage Integration**: Secure connection between Snowflake and S3 via IAM
- **External Stage**: Points to S3 bucket
- **Snowpipe**: Auto-ingests new S3 files into Snowflake table
- **Snowflake DB**: Final data warehouse for analytics

## Flow
1. EventBridge triggers Lambda every scheduled interval
2. Lambda fetches weather from OpenWeatherAPI
3. Data saved to DynamoDB + S3
4. DynamoDB Stream triggers 2nd Lambda
5. 2nd Lambda writes processed data to S3
6. S3 sends event notification to SQS
7. Snowpipe listens to SQS → auto-loads data into Snowflake
8. Data available in Snowflake for analytics
